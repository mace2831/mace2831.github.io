Algorithms and Data Structures

In this artifact I further refined the AnimalShelter project two from CS-340 by adding a Trie data structure to store the names of the animals in the database and offer suggestions based off of user input in the search bar. I wanted to add this because I have never worked with a Trie data structure and I wanted to add some more functionality and user friendliness to the web application. I met all the course objectives I planned to meet with this enhancment. The first enhancement I didn’t finish the authentication portion of the SPA, but I plan on adding that in on the third enhancement for the database. This process was a bit challenging as I have never implemented a Trie data structure before. I also had to learn how to create a new endpoint within the Express application in order to access the information stored in the Trie structure. When I implemented the onClick feature that takes the user to the seleceted animal’s page I had a number of isseus with trying to use the animal name rather than the rec_num. I ended up having to add in functionality to the autocomplete.js function to get the rec_num with the animal name in order to properly route the user to that selected animal’s page. There were a number of tutorials and articles on w3schools and GeeksforGeeks that were rather helpful.
