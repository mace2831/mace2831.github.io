Databases

In this artifact I further refined the AnimalShelter project two from CS-340 by completing the single page administrator application that limits database manipulation to only authorized users. I added this artifact into my ePortfolio to show my ability to create a secure web application that protects data in the database while allowing users to safely view the information. This enhancement clearly showcases my security mindset by anticipating security breaches and mitigating vulnerabilites. The Trie data structure from the second enhancement that is used for the autocomplete function prevents any sort of injection attack to the database as the interaction with the database is done without user input. When taking in the username from the user Mongoose sanitize is used to sanitize the input to remove threats from injection attacks. On the administration page the buttons that allow the user to manipulate the data in the database only become visible when a valid user and JWT certificate are verified. The course outcomes that align with this enhancement are 3 and 5 and demonstrated my ability to design and evaluate computing solutions that sove a given problem as well as show my security mindset by anticipating exploits and mitigate those vulnerabilities through the implementation of data sanitization and user authorization and authentication. This process was easier that adding the autocomplete functionality because I have created a single page application administrator page before. Adding in the data sanitization through Mongoose sanitize was an easy process, but a good learning experience.
